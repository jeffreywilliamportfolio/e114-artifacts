Hypothesis testing is a statistical method used to make decisions about a population based on sample data. It allows researchers to determine if observed effects are likely due to chance or if they represent a real phenomenon.

Here is a step-by-step breakdown of how it works, focusing on the core components you requested.

### 1. The Core Framework: Null vs. Alternative
Before calculating anything, you must define two competing hypotheses about the population parameter (e.g., the mean, proportion, or variance).

*   **The Null Hypothesis ($H_0$):** This represents the "status quo" or the assumption of "no effect." It states that any observed difference in the data is due to random chance.
    *   *Example:* "The new drug has no effect on blood pressure compared to the placebo."
*   **The Alternative Hypothesis ($H_1$ or $H_a$):** This represents the researcher's claim or the effect they are trying to detect. It states that there is a real difference.
    *   *Example:* "The new drug lowers blood pressure compared to the placebo."

### 2. The Null Model
The **null model** is the theoretical probability distribution that describes what the data would look like if the Null Hypothesis ($H_0$) were true.

*   **Purpose:** It serves as the baseline for comparison.
*   **Construction:** Based on $H_0$, we assume a specific distribution (e.g., a Normal distribution with a mean of 0, or a t-distribution).
*   **Visualization:** Imagine a bell curve centered at the value predicted by $H_0$. The area under this curve represents the probability of observing different sample outcomes purely by chance.

### 3. The Test Statistic
Once you collect your sample data, you calculate a **test statistic**. This is a standardized value that measures how far your sample result deviates from the null model.

*   **Formula:** Generally, it follows the structure:
    $$ \text{Test Statistic} = \frac{\text{Observed Value} - \text{Expected Value (under } H_0)}{\text{Standard Error}} $$
*   **Common Types:**
    *   **$z$-score:** Used when population variance is known or sample size is large.
    *   **$t$-score:** Used when population variance is unknown and sample size is small.
    *   **$\chi^2$ (Chi-square):** Used for categorical data.
*   **Interpretation:** A test statistic close to 0 suggests the data fits the null model well. A large absolute value (positive or negative) suggests the data is far from what $H_0$ predicts.

### 4. The P-Value
The **p-value** is the probability of obtaining a test statistic at least as extreme as the one observed, assuming the null hypothesis is true.

*   **Small P-value (e.g., $< 0.05$):** Indicates that the observed data is very unlikely under the null model. This suggests we should reject $H_0$.
*   **Large P-value:** Indicates the data is consistent with the null model. We fail to reject $H_0$.
*   **Note:** The p-value is *not* the probability that $H_0$ is true; it is the probability of the data given $H_0$.

### 5. Error Rates: Type I and Type II
Because we are making decisions based on probability, we are never 100% certain. There are two types of errors:

#### Type I Error (False Positive)
*   **Definition:** Rejecting $H_0$ when it is actually true.
*   **Probability ($\alpha$):** This is the **Significance Level**. It is set by the researcher before the test (commonly 0.05 or 5%).
*   **Meaning:** If $\alpha = 0.05$, there is a 5% chance you will conclude there is an effect when there actually isn't one.

#### Type II Error (False Negative)
*   **Definition:** Failing to reject $H_0$ when $H_1$ is actually true.
*   **Probability ($\beta$):** This is usually harder to calculate directly.
*   **Power ($1 - \beta$):** The probability of correctly rejecting $H_0$ when it is false. High power (usually > 0.80) is desirable.
